# AIShortHub 管理后台操作手册（Phase 13）

## 1. 商业模型运营要点
平台收入结构已调整为：
1) 广告收入
2) 创作者服务订单
3) Viewer 订阅
4) 单次解锁（整剧/单集）
5) 低抽成

运营评估应避免仅看抽成，需联合评估广告与服务收入质量。

## 2. 创作者审核流程
1. 检查 Trailer 是否完整（标题/封面/CTA/时长）。
2. 检查 Main Episodes 元数据（排序、试看/收费、字幕、上架时间）。
3. 按 QC 提示核验素材质量。
4. 审核通过后进入发布与结算链路。

## 3. 收益面板解读
重点看：
- 广告分成、订阅池分成、整剧/单集销售
- 服务成本与平台抽成
- 净收入、待结算、已打款

建议对“净收入异常下降”触发人工巡检（内容质量、字幕、分辨率、支付异常、广告供给）。

## 4. 退款与客服协同
- Viewer：低摩擦取消，已开始周期默认不按比例退款。
- Creator：上传/审核/权益消耗后不可退款。
- Add-on：未开工可退款，开工后默认不可退款。

涉及争议订单时，记录最终处置为退款、拒绝、或平台 credit。
